How it works

From POST /v1/messages to delivered, in seconds.

STEP 1

Detect

We scan your message for products, events, dates, URLs, and other structure.

STEP 2

Validate

Phone numbers normalized. Image URLs health-checked. Broken content stripped.

STEP 3

Optimize

Best format selected — text, rich card, carousel — based on what will actually render.

STEP 4

Comply

TCPA scanning, STOP/START keyword handling, consent checks — before it leaves.

STEP 5

Deliver

Carrier-direct RCS delivery. We handle the agent registration, the trusted sender badge, everything.

STEP 6

Fall Back

Non-RCS devices get clean SMS with inline tappable URLs — no empty cards, no broken experiences.

STEP 7

Forward Replies

Inbound taps and text replies hit your webhook or MCP server in real time.

Be lazy or be precise

Two paths. Same API.

Send plain text and let our AI extract structure — or pass explicit items and control every detail. Same endpoint, same response shape, same delivery guarantees.

  • Raw text path — perfect for AI agents, chatbots, and LLM-driven flows
  • Structured path — perfect for systems that already have product data or event records
  • MCP server — drop-in for Claude Code, Cursor, and other agent tooling
Try both paths live →
